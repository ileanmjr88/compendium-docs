## Origin
This project came from the frustration of setting up a new development environment every time I switched between my MacBook and my Linux computer, and losing the first few days at every new job installing the tools needed to work with the codebase. The documentation was often stale, since the tool setup had changed since the last update.

Compendium came out of that frustration. The goal: set up a project across different computers with the same pinned compilers, runtimes, and supporting tools, easily and without headaches.

## What Compendium is
Compendium lets you declare your project's compilers, runtimes, and tools in a single config file, pinned to specific versions. One command installs everything into an isolated per-project environment.

## How the project is organized
The project has two components:
- [compendium](https://github.com/ileanmjr88/compendium) — the CLI (Go, GPL v3). Reads your config, queries the registry, installs the environment.
- [compendium-registry](https://github.com/ileanmjr88/compendium-registry) — a curated index of tools with prebuilt binaries for macOS and Linux (CC0, fork it freely).

## Scope
Compendium manages per-project development environments. It is not a container runtime, not a CI system, and not a replacement for Docker when you need true OS-level isolation. If your project needs a database or a message broker running locally, use Docker Compose alongside Compendium, they solve different problems.
